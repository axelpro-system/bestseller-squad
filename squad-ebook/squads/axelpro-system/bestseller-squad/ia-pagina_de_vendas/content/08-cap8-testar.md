# Capítulo 8: Como Validar e Testar Sua Página

## Por que testar é essencial

Você pode ter o melhor produto do mundo, mas se sua página não converte, ninguém compra.

Testar não é opcional. É obrigatório.

A boa notícia: com IA, você pode testar versões em minutos — não dias.

---

## Testes Essenciais (Obrigatório)

### 1. Teste de Velocidade

**Meta:** Página carrega em menos de 3 segundos

**Ferramentas:**
- Google PageSpeed Insights (grátis)
- GTmetrix (grátis)
- Pingdom (grátis)

**O que otimizar:**
- Imagens: comprima (TinyPNG)
- Código: minify CSS/JS
- Hospedagem:inga mais rápida

---

### 2. Teste Mobile

**Meta:** Funciona perfeitamente no celular

**Verificações:**
- [ ] Layout responsivo
- [ ] Botões tocáveis (mínimo 44px)
- [ ] Texto legível sem zoom
- [ ] Imagens não cortadas
- [ ] Formulário funciona

**Ferramentas:**
- Chrome DevTools (F12 → Device Mode)
- BrowserStack (pago)

---

### 3. Teste de Usabilidade

**Meta:** Qualquer pessoa consegue usar

**Tarefas:**
1. Peça para 3 pessoas usarem
2. Cronometre quanto tempo levam
3. Observe onde Param
4. Anote perguntas que fazem

**Problemas comuns:**
- Botão difícil de encontrar
- Formulário confuso
- Informações contraditórias

---

## Testes de Copy (Recomendado)

### Teste A/B de Headlines

Crie 2 versões:
- Versão A: [Headline original]
- Versão B: [Headline alternativa]

**Como testar:**
1. Use Google Optimize (grátis) ou:
2. Carrd Pro permite teste básico

**Métricas:**
- Taxa de clique (CTR)
- Tempo na página

---

### Teste de Imagens

Teste diferentes:
- Com/sem imagem na hero
- Foto pessoa vs illustration
- Video vs static

**Perguntas:**
- A imagem comunica rápido?
- Está profissional?
- Relevante para o produto?

---

## Otimização Contínua

### O ciclo de melhoria

```
1. Publicar página
2. Coletar dados (7-14 dias)
3. Identificar problemas
4. Criar hipótese
5. Testar mudança
6. Medir resultado
7. Repetir
```

### Métricas para acompanhar

| Métrica | Meta Inicial | Bom | Excelente |
|---------|--------------|-----|-----------|
| Taxa de conversão | 1% | 3% | 5%+ |
| Tempo na página | 30s | 60s | 120s+ |
| Taxa de rejeição | <70% | <50% | <30% |
| CTR do CTA | 2% | 5% | 8%+ |

---

## Ferramentas de Análise

### Google Analytics 4 (Grátis)

1. Crie conta GA4
2. Pegue ID de measurement
3. Adicione script na página
4. Configure Goals (conversões)

**Eventos para medir:**
- Page view
- CTA click
- Scroll depth
- Form submission

### Hotjar / Microsoft Clarity (Grátis)

**O que mostra:**
- Heatmaps (onde clicam)
- Screen recordings
- User journey

**Ideal para:**
- Entender comportamento
- Identificar problemas UX

## Checklist Final de Lançamento

### Antes de publicar:
- [ ] Copy revisado 3x
- [ ] Sem erros de português
- [ ] Imagens otimizadas
- [ ] Links funcionando
- [ ] mobile funfa
- [ ] Velocidade < 3s

### Na semana 1:
- [ ] Monitorar acessos
- [ ] Testar checkout
- [ ] Pedir feedback
- [ ] Ajustar se necessário

### Na semana 2-4:
- [ ] Analisar métricas
- [ ] Identificar gargalos
- [ ] Testar melhorias
- [ ] Aumentar tráfego

---

## Quando Refazer a Página

Sinais de que precisa mudar:

1. **Taxa de conversão < 1%** por mais de 30 dias
2. **Feedback negativo consistente**
3. **Produto mudou** (novo público, preço, etc)
4. **Passou mais de 6 meses** (visuais desatualizam)
